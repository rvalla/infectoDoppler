iD: browsers

recomended browsers

The app that record the players is written in p5js. I tested it in several browsers and I is working on Firefox, Opera, Brave, Chrome and Safari on desktops. In Android works on Brave, Opera and Chrome but is not working in Firefox. In iOS I tested in Safari and works correctly although it download a .dms file and not a .wav.

input level

If you note that the audio file you download is so loud or saturated you can configure the audioIn input level changing the url parameters. By default I set the game url with "?mic=false" parameter. You can pass a true value or simply remove it to enter the audioIn configuration before the game. Yoy can test it here.
This tool is not working in Firefox. If it is your favorite browser simply set audioIn level passing a number in the url (adding ?amp=n). Remember that this number has to be between 0 and 1 (0 <= n <= 1).
